01

Change Orders

A change order is work that is added to or deleted from the original contract, which alters the amount and/or completion date. This may force a new project to handle significant changes to the current project.

02

Users

Users may be placed on teams, which in turn are placed on to projects. Teams can then be used as distribution groups and have actions applied to them, such as, sent text/SMS messages, emailed, given a role and more.

03

List

List allow you to keep track of Company, Project or Team wide events. With proper permissions one can create events for the Company -or- the Project. The Team list anyone can post events to for all others.

04

Alerts / Messages

All software needs the ability to communicate with the end-user success and error messages. We capture all error messages and log them. This allows us to know where our bugs are and proactively fix them.

05

Infinite-Scroll

Infinite scrolling is a web-design technique that loads content continuously as the user scrolls down the page, eliminating the need for pagination. Popular websites like Twitter and Facebook rely on infinite-scrolling.

06

Themes & Settings

A theme is the set of colors that will be applied to the Angular Material components. You can also choose a route-transition. These transitions are then used when switching routes.

07

Large Tables

This shows a way to handle large tabular data. The table's header and first column are fixed in place; that is they don't scroll like the other columns do. The user may also choose which columns to include/exclude.

08

Recursion

This example uses recursion. There is an unknown number of line-items. Each line-item can be nested to the nth-level. So we use recursion to create a nested hierarchy of line-items and all their children.

09

Tree & Stuff

This is a hierarchical-tree-view that allows users to select items. The tree can have nth-level of children. Once selected the item is brought over into a grid. The heirarchy of parent/child/grand-children is maintained.
